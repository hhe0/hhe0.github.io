CSS那些事(一)：花落知多少——CSS正传

发表于 2018-11-12   |   更新于 2019-05-07 | 分类于 CSS读书笔记系列 | | 阅读次数

所谓的CSS，是由W3C(万维网联盟)的CSS工作组创建和维护的，不需要编译，增强控制网页样式并允许将样式信息与网页内容分离，是可以直接由浏览器执行的一种标记性语言，或者说它是一种浏览器解释性语言。

网页美容师——认识CSS

CSS的作用

  • 修饰页面文本、图片等页面元素，避免使用不必要的HTML元素；
  • 更有效地控制页面结构、页面布局；
  • 提高开发和维护效率。

CSS的基本结构

  • 选择符：告诉浏览器该样式将匹配于页面中的哪些对象；
  • 声明：告诉浏览器怎样去渲染页面中的与选择符相匹配的对象；
  • 属性：以一个单词的形式出现，并且都是CSS约定的而不是自定义的；
  • 属性值：取决于属性。

CSS结构的特点

  • 声明都是紧跟着选择符，并被花括号包含着；
  • 每个声明的属性跟属性值之间都是用冒号隔开、分号结束的；
  • 花括号及分号前后的空格可有可无；
  • 属性值名称过长并带有空格，一定要将属性值用引号包含，如”sans serif”。

CSS中的注释

  • 提倡合理使用CSS注释，但过度使用将适得其反，使CSS文件增多不必要的字节。

网页勾魂八计——掌握CSS选择符

通配符选择符

类选择符

包含选择符

  • 又称为派生选择符或者后代选择符。

子选择符

  • 并非所有浏览器都支持；
  • 定义的是直接子元素的样式。

相邻选择符

  • 定义的样式是紧紧相邻的两个标签元素，且二者有相同的父元素。

属性选择符

  • 属性选择符并不是所有浏览器都支持。

ID选择符

  • 一个页面不要建议不要使用同名ID，避免出现JS错误。

选择符的组合关系

  • 针对性使用类选择符/ID选择符、选择符群组及选择符组合。

针对性使用类选择符/ID选择符

选择符群组

  • 选择符之间的关系为并列关系。

选择符组合

  • 选择符之间的组合为递进关系；
  • 目标选择符为最后一个选择符。

莫须有？——伪类与伪对象

伪类

  • 伪类可以让用户在使用页面的过程中增加更多交互效果，而不必去使用过多的JS来辅助完成。

伪对象

剪不断理还乱——善处选择符之间的关系

选择符的覆盖

  • 常用于在样式中将所有标签的内外补丁定义为0，然后根据实际页面需求而再次定义内外补丁的间距。

选择符z的继承

  • 子承父业，但是并不是所有的后代标签都会继承父级元素的CSS属性；
  • 听起来有点像嫌疑人X的献身2333。

选择符的权重值优先级别

CSS的优先级

  • 标有!important关键字声明的属性；
  • HTML中的CSS样式属性；
  • 作者编辑的CSS文件样式属性。以选择符的组合方式来决定优先级别，相同的选择符组合方式将根据作者定义CSS样式的先后次序而决定优先级别；
  • 用户设置的样式；
  • 浏览器默认的样式。

选择符的优先级

  • 标签选择符、伪类及伪对象：优先级积分为1；
  • 类选择符、属性选择符：优先级积分为10；
  • ID选择符：优先级积分为100；
  • style属性：优先级积分为1000；
  • 其他选择符，如通配符选择符等：优先级积分为0；
  • 根据以上优先级积分，统计CSS样式表中出现的每个选择符个数，再相加，最终得出的积分数就是某个选择符的优先级别。

英雄救美——让CSS拯救HTML

把CSS引入到HTML页面中

  • 直接写在标签元素的属性style中，通常称为行间样式；
  • 将样式写在标签之内，通常称为内嵌样式表；
  • 通过方式外链CSS样式文件，通常称为外联样式表；
  • 通过@import关键字导入外部CSS样式文件，通常称为导入样式表。

样式表的规划与维护管理

  • 一般分为页面的全局定义及处理页面基本布局的CSS样式文件和处理细节方面的CSS样式文件；
  • 一般选择一个大型的CSS文件而不是多个CSS小文件；
  • 未发布站点前的注释可以在正式发布后进行删除。
