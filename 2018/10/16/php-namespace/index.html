PHP基础——命名空间

发表于 2018-10-16   |   更新于 2019-05-07 | 分类于 PHP随笔系列 | | 阅读次数
字数统计: | 阅读时长 ≈

命名空间是PHP乃至于面向对象中的一个非常重要的概念。

什么是命名空间

  • 命名空间是一种封装事物的方式。事物具体指的是代码中的类/接口/函数/常量。

命名空间的用途

  • 命名空间是用来解决用户编写的代码与PHP内部类/函数/常量或者第三方类/函数/常量之间的名字冲突。

命名空间怎么使用

命名空间的定义

  • 命名空间的声明不能以PHP/php开头，这些命名空间被保留做语言内核使用；
  • 命名空间必须在所有代码之前（除declare关键字）进行声明；
  • 允许同一个命名空间定义在多个文件中。

在一个文件中声明多个命名空间

  • 不推荐使用！！！一般用于将多个PHP脚本合并在一个文件中；
  • 定义多个命名空间，尽量使用大括号语法，明确其命名空间的作用域；、
  • 如果文件中没有声明命名空间，则其表示为全局的；
  • 当一个文件中既存在全局的命名空间也存在非全局的命名空间，则他们必须使用括号语法，且全局的命名空间也需要进行namespace{}声明。

命名空间内元素的引用方式

  • 可以有三种方式：非限定名称、相对限定名称和完全限定名称；
  • 注意访问任何全局类，函数或变量，都可以使用完全限定名称，例如\Exception。

命名空间与动态语言特征

  • 必须使用完全限定名称的引用方式。

namespace关键字与__NAMESPACE__魔术变量

  • __NAMESPACE__保存当前命名空间的字符串，如果为全局的命名空间不包含任何其他的命名空间，则字符串为空。常用于动态创建名称；
  • namespace用于显示访问当前命名空间或者子命名空间中的元素，它等价于类中的self操作符；

使用别名/导入

后备全局函数/常量

  • 类名称总是解析到当前命名空间（所访问的命名空间）中的类名称，在访问系统内部或不包含在命名空间中的类名称时，必须使用完全限定名称；
  • 如果在命名空间中不存在该函数或变量，PHP会退而使用全局空间中的函数或常量；

名称解析规则

  • 对完全限定名称的函数，类和常量的调用在编译时解析；
  • 所有的非限定名称和限定名称（非完全限定名称）根据当前的导入规则在编译时进行转换；
  • 在命名空间内部，所有的没有根据导入规则转换的限定名称均会在其前面加上当前的命名空间名称；
  • 在命名空间内部对非限定名称或限定名称类（非完全限定名称）的调用是在运行是解析的。
