【深入PHP：面向对象、模式与实践】读书笔记

发表于 2018-10-13   |   更新于 2019-05-07 | 分类于 PHP读书笔记系列 | | 阅读次数

在本章中，通过介绍PHP的核心面向对象特性奠定一些基础知识，以便后面深入分析对象与设计。

类和对象

  • 如果类是生成对象的模板，那么对象是根据类中定义的模板所构造的数据；
  • 对象可以被说成是类的『实例』，它是由类定义的数据类型。

编写第一个类

第一个对象（或两个）

  • 把对象直接当做字符串输出会出错，可以自定义魔术方法__toString()；
  • 通过传递对象给var_dump()，我们可以获得它所包含的有用信息——每个对象的内部标志符（#号后面的数字）。

设置类中的属性

  • 类可以定义被称为属性的特定变量。属性也被称为成员变量，用来存放对象之间互不相同的数据；
  • 调用类、函数或方法的代码通常被称为该类、函数或方法的客户端或者客户端代码；
  • PHP并没有强制所有的属性都要在类中声明。我们可以动态地增加属性到对象，但这种给对象赋属性的方法在面向对象编程中并不是一个良好的做法，我们基本上不使用。

使用方法

  • 和函数不同的是，方法必须在类中声明；
  • 如果在方法声明中省略了可见性关键字，那么方法将被隐式声明为public。

创建构造方法

  • 构造方法可以用来确保必要的属性被设置，并完成任何需要准备的工作。

参数和类型

  • 在PHP中，定义方法和函数时不需要指定参数的数据类型；
  • 参数可以是任何类型给我们提供了很大的灵活性。你可以创建灵活响应不同数据类型的方法，根据不同的环境调整其功能；
  • 当类方法希望一个参数必须是某种数据类型（而不是其他类型）时，这种灵活性可能会引起代码的歧义。

基本类型

  • 处理方法和函数的参数时，检查变量的类型尤其重要；
  • 数据类型和类的一个关键区别：定义一个类也就定义了一个类型，但是一个类型可以用于描述一个家族的众多类。

基本类型和PHP的类型检查函数

类型检查函数 类型
is_bool() 布尔型
is_integer() 整型
is_double() 双精度型
is_string() 字符串
is_object() 对象
is_array() 数组
is_resource() 资源
is_null() NULL

获得提示：对象类型

  • 可以将类或者数组放在参数前来约束方法参数；
  • 有了参数的类型提示，我们不再需要在使用参数前对其进行类型检查。虽然自动类型检查是防止bug的极佳途径，但我们要知道类型提示是在运行时才生效的；
  • 类型提示不能用于强制规定参数为某种数据类型，比如字符串和整型。如果要处理基本数据类型，在方法体中要使用is_int()这样的类型检查函数。

继承

  • 继承是从一个基类得到一个或多个派生类的机制。

继承问题

  • 我们希望能为每种类型提供一个单独的工具，但同时希望提供公用的功能来避免重复，又允许每种类型处理不同的方法调用。这正是我们使用继承的原因。

使用继承

  • 创建继承树的第一步是找到现有基类元素中不适合放在一起，或者不需要进行特殊处理的类方法。

构造方法和继承

  • 在子类中定义构造方法时，需要传递参数给父类的构造方法，否则你得到的可能是一个构造不完整的对象；
  • 每个子类都会设置自己的属性前调用父类的构造方法。基类现在仅知道自己的数据。子类一般是父类的特例。我们应该避免告诉父类任何关于子类的信息，这是一条经验规则。

调用被覆写的方法

  • PHP使用parent::__contruct()来调用父类的构造方法。

public、private、protected：管理类的访问

  • 在任何地方都可以访问public、private或protected；
  • 只能在当前类中才能访问private方法或属性，即使在子类中也不能访问；
  • 可以在当前类或子类中访问protected方法或属性，其他外部代码无权访问。

访问方法

  • 当客户端程序员需要使用类中保存的值时，通常比较好的做法是不要允许直接访问属性，而是提供方法来取得需要的值，这样的方法被称为访问方法，也可称为getter和setter。
