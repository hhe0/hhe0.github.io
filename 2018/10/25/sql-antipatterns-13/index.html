SQL反模式(十三)：对未知的恐惧

发表于 2018-10-25   |   更新于 2019-05-07 | 分类于 SQL读书笔记系列 | | 阅读次数
字数统计: | 阅读时长 ≈

使用NULL来表示任意类型的悬空值。

目标：辨别悬空值

  • 你可以在添加一条记录时，使用NULL代替哪些还不确定的值；
  • 一个给定的列如果没有合适的值，可以在对应的行中使用NULL；
  • 当传入参数无效时，一个函数的返回值也可以是NULL；
  • 在外联结查询时，NULL被用来当做未匹配的列的占位符。

反模式：将NULL作为普通的值，反之亦然

  • SQL将NULL当做一个特殊的值，不同于0、false或者空字符串（部分数据库例如Oracle可能不同）。

在表达式中使用NULL

  • NULL和0是不同的；
  • NULL和空字符串也是不一样的。将一个字符串和标准SQL中的NULL联合起来的结果还是NULL；
  • NULL和FALSE也是不同的。AND、OR和NOT这三个布尔操作如果涉及NULL，其结果要让很多人困惑。

搜索允许为空的列

  • WHERE子句的筛选逻辑是当其条件的返回值为TRUE时选择该记录，但和NULL比较永远得不到TRUE，相应地，返回的是『未知』。无论比较的逻辑是相等还是不等，返回的结果还是『未知』，当然『未知』不等于TRUE。

在查询参数中使用NULL

  • 在参数化的SQL查询表达式中使用NULL进行查询时，碰到使用NULL作为普通值的列也非常地不方便，无法将NULL作为参数传入。

避免上述问题

  • 悬空的值应该用NULL来表示。

如何识别反模式

  • 我们怎么将这一列中没有值的列取出来？
  • 我能在数据库中看到用户的名字，但是在程序显示的时候，名字就是空的；
  • 这份报告中这个项目的总工作时间按仅包括了很小一部分我们设定了优先级的Bug；
  • 现在的情况是，我们不能再使用之前在表中用来表示未知的那个字符串了，因此我们要开个会讨论改用哪个新的特殊值，再评估一下开发时间以及数据转换的时间。

合理使用反模式

  • 有一种情况可以将NULL视为普通值，那就是导入或者导出数据的时候；
  • 类似地，用户不能直接输入一个NULL。程序的输入端可能会使用一些特殊处理来引导用户输入NULL；
  • 如果需要支持多个不同的悬空值时，NULL也不起作用。

解决方案：将NULL视为特殊值

  • 大多数使用NULL的问题都是源自于对SQL的三值逻辑的误解。

在标量表达式中使用NULL

  • NULL的含义表示『未知』，与『未知』运算的结果都是『未知』。

在布尔表达式中使用NULL

  • NULL既不是TRUE也不是FALSE。

检索NULL值

声明NOT NULL的列

  • 对于一个在逻辑上没有默认值的列来说，声明一个NOT NULL约束时很合理、很常见的。

动态默认值

  • 在特定的请求时对一个给定的列临时设置一个默认值的方法。Mysql中存在ISNULL函数实现这个功能。
