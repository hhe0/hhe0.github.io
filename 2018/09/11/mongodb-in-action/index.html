MongoDB基础知识介绍。

全新Web数据库

MongoDB是为快速开发互联网Web应用而设计的数据库系统。其数据模型和持久化策略就是为了构建高可读/写吞吐量和高自动灾备伸缩性的系统。

为互联网而生

MongoDB的设计目标就是极简、灵活，作为Web应用栈的一部分。

MongoDB键特性

本节介绍了文档数据模型、MongoDB对此数据模型的高效处理特性以及MongoDB的主从复制和水平扩展策略。

文档数据模型

无schema模型的优点
  • 应用程序的代码强制数据结构而不是数据库。在频繁修改数据定义的时候这可以加速应用程序开发；
  • 无schema模型允许用户使用真正的变量属性来表示数据。

ad hoc查询

  • 系统支持主动查询模式(ad hoc queries)是指不需要事先定义系统接收何种查询；
  • 键-值存储数据库牺牲丰富的查询功能来换取更简单的伸缩模型，这种类型的数据库只支持一个类型的查询：键key；
  • MongoDB的设计目标之一就是保留大部分关系型数据库的功能。

索引

MongoDB中的索引就是使用了B-树（平衡树）数据结构。B-树索引也大量使用与许多关系型数据库中，对于不同的查询进行优化，包括范围查询和条件字句查询。

复制

  • MongoDb提供了数据库复制特性，叫做可复制集合；
  • 可复制集合在多个机器上分布式存储数据，在服务器或者网络出错时，实现数据冗余存储和自动灾备；
  • 复制还可用于伸缩数据库读操作，通过分散读取压力到可复制集群中的服务器来实现；
  • 可复制集合的主节点同时接受读/写操作，但是从服务器只能进行读操作；
  • 通过复制实现自动化灾备过程：如果主节点失败，则集群中会选择一个从节点，并自动提升为主节点。当之前的主节点回归时，

加速与持久化

  • 写入速度可以理解为数据库在给定的时间内插入、更新和删除的容量；
  • 持久性指的是这些写操作被永久保存的保证级别；
  • 设计MongoDb的目标就是让大家可以保持速度与持久性平衡，但是对于重要的数据，强烈推荐使用安全设置。

伸缩

  • 垂直扩展：通过组合添加更快的磁盘、更多内存以及更强的CPU来解除数据库性能瓶颈；
  • 水平扩展：在多台机器上分布式存储数据库，而不是提升单个节点的配置。

核心服务和工具

核心服务器

核心服务器通过名为mongod的可执行文件运行。

JavaScipt shell

MongoDB命令行工具是一个基于JavaScript的数据库操作和管理工具。

数据库驱动

  • 驱动是应用程序用来与MongoDB服务器通信的代码；
  • 所有驱动都保护查询功能、搜索结果、写入数据和运行数据库命令；
  • 所有的驱动都实现了相似的方法来保存数据到集合中，但是文档的表示类型都是基于自己语言的最原始的类型定义。

命令行工具

  • mongodump 和 mongorestore
    备份和恢复数据库的工具；
  • mongoexport 和 mongoimport
    导入或者导出JSON、CSV、TSV格式的数据；
  • mongosniff
    一个用于查看发送给数据库命令的嗅探工具；
  • mongostat
    轮询MongoDB，提供有帮助的状态信息：包括每秒的操作数（增、删、改、查等），分配虚拟内存的数量，以及服务器的连接数量；
  • mongotop
    轮询MongoDB，并且显示它在每个集合里花费的读取和写入数据的时间总数；
  • mongoperf
    帮助我们了解MongoDB实例磁盘操作的情况；
  • mongooplog
    展示MongoDB操作日志里的信息；
  • Bsondump
    把BSON文件转换为人类可读的格式，包括JSON。

为什么是MongoDB?

设计目标

易于伸缩，存储丰富的数据结构，并且提供复杂的查询语言。

使用场景

  • MongoDB是做Web应用、分析应用的首要数据库；
  • 比较容易存储无schema数据，也就是弱数据结构的数据；
  • 存储无法事先知道数据结构的数据；
  • 进行敏捷开发；
  • 适用于分析和日志；
  • 将MongoDB作为可以支持更多查询功能的缓存使用，或者直接与缓存层集成到一起。

提示和限制

  • MongoDB通常用于64位系统，超过一定的内存后会涉及到磁盘的换页导致速度变慢；
  • MongoDB的查询语句与SQL差别较大；
  • MongoDB运行大规模集群有维护的成本。

命令如何执行

无论提供什么功能，所有的数据库命令实现都有个共同点，就是它们在一个叫做$cmd的虚拟集合上实现查询。

  • 只输入方法名，不加上括号，可以显示方法对应的源代码实现。
