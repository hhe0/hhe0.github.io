【MongoDB权威指南】读书笔记(二)

发表于 2018-09-07   |   更新于 2019-05-07 | 分类于 MongoDB读书笔记系列 | | 阅读次数

介绍如何对MongoDB文档进行增、删、改、查。

创建、更新和删除文档

介绍如何对MongoDB中的文档进行创建、更新和删除操作，以及如何为这些操作选择合适的安全级别和速度。

插入并保存文档

批量插入

  • 只有需要将多个文档插入到一个集合时，批量插入的方式才有用；
  • 不能在单次请求中将多个文档批量插入到多个集合中；
  • 如果在执行批量插入的过程中，有一个文档插入失败，那么在这个文档之后的所有文档都会插入失败。可以使用continueOnError选项忽略该错误继续执行。

数据检查

  • 插入数据时，MongoDB只对数据进行最基本的检查；
  • 检查文档的基本结构（比如文档大小不超过16MB）；
  • 如果没有”_id”字段，就自动增加一个。

删除文档

清空集合

删除集合

采用remove方法删除结合只会清空集合中的文档，不会删除集合本身。

更新文档

更新操作是不可分割的，具有原子性：若是两个更新同时发生，先到达服务器的先执行，接着执行另一个。所以，两个需要同时进行的更新会迅速接连完成，此过程不会破坏文档：最新的更新会取得“胜利”。

更新整个文档

如果需要更新整个文档，那么我们可以使用”_id”作为查询条件，防止结果有多个造成更新失败。

更新文档的一个部分

$set修改器

$set用来指定一个字段的值。如果这个字段不存在，则创建它。

$inc修改器

专门用来增加（和减少）数字。$inc只能用于整型、长整型或双精度浮点型的值，不能用于字符串、数组或其他非数字的值。

数组修改器

数组不仅可以通过索引进行引用的列表，而且还可以作为数据集（set）来用。

a. 向数组/数据集中添加元素

使用$each实现多次$push

使用$slice保证数组的大小，$sort用来清理元素前的排序

将数组作为数据集使用可以保证数组内的元素不会重复

b. 删除数组/数据集中的元素

可以把数组视为栈（从数组末端删除）或者队列（从数组首端删除）

数组操作符只能用于包含数组值的键。例如：不能将一个整数插入数组，也不能将一个字符串从数组中弹出。要修改标量值，得使用$set或者$inc。

c. 基于位置的数组修改器

我们通过位置或者定位操作符对于数组的一部分进行操作

修改器速度

当由于某种修改增大文档的大小可能会导致文档的顺序重排，比如放到集合的尾部，这样会导致集合的填充因子变大（磁盘中文档的空白部分变多）。当文档不再发生移动时，填充因子的值会缓慢降低。

Insert与Update的结合：Upsert

根据查询条件找到文档则更新，找不到则创建一个新的文档。便于对文档进行统一处理的操作。

更新多个文档

将update的第四个参数设为true即可。

返回被更新的文档

使用findAndModify来返回被更新的文档。

写入安全

  • 两种最基本的写入安全机制是应答式写入和非应答式写入。应答式写入是默认的方式：数据库会给出响应，告诉你写入操作是否执行成功。非应答式写入不返回任何响应，所以无法知道写入是否成功；
  • 通常来说，应用程序应该使用应答式写入。但是，对于一些不重要的数据（例如日志或者批量加载数据），可以使用非应答式写入。

查询

介绍如何对集合中的文档查询以及如何使用数据库游标的相关知识。

find查询

查询条件

  • 比较操作符有：$lt(<)、$lte(<=)、$gt(>)和$gte(>=)；
  • 不等于使用$ne；
  • OR查询有两种方式：$or、$in；
  • $not常配合正则表达式使用。

$or可以在任何情况下正常工作，而查询优化器可以更高效地处理$in，优先使用$in。

使用普通的AND型查询时，总是希望尽可能用最少的条件来限定结果的范围。OR型查询正相反：第一个条件应该尽可能匹配更多的文档，这样才是最高效的。

特定类型的查询

null

比较特殊，需要格外小心。

正则表达式

任何PCRE支持的正则表达式语法都能被MongoDB接受。另外，正则表达式可以匹配自身。

查询数组

尝试把数组看成多个键相同的键值对。

a. 单个元素查询数组

b. 多个元素查询数组

c. $size

$size不能与其他查询条件（比如$gt）组合使用，换句话说，只能表达等于某个size的范围。如果有查询size类型范围的需求，考虑维护一个数组大小的键值对。

d. $slice操作符

有一点比较特殊：除非特别声明，否则使用$slice时将返回文档中的所有键。

e. 数组与范围查询的问题

对于标量来说，文档必须要与给出的范围完全匹配；而对于数组来说，可能只要数组中存在一些元素满足其中的条件即可。

查询内嵌文档

有两种方法可以查询内嵌文档：查询整个文档，或者只针对其键/值对进行查询。

点表示法要求文档的键不能存在.号，通常在将URL作为键保存时会有这样的问题，可以通过字符替换的方式解决。

$where查询

为了安全起见，应该严格限制或者消除$where语句的使用。$where主要用于表达一些键值对无法表达的场景。

游标

数据库使用游标返回find的执行结果。通过客户端的游标可以限制结果的数量、略过部分结果、根据任意键按任意顺序的组合对结果进行各种排序等操作。

对混合类型的键的排序顺序：

  • 最小值；
  • null；
  • 数字（整型、长整型、双精度）；
  • 字符串；
  • 对象/文档；
  • 数组；
  • 二进制数据；
  • 对象ID；
  • 布尔型；
  • 日期型；
  • 时间戳；
  • 正则表达式；
  • 最大值。

避免使用skip略过大量结果

需要skip的数据很多时，速度很慢，效率很低。

一些常见应用场景

  • 使用其他方式进行分页而不是skip：找到上一页最后一条数据的特征进行过滤然后分页，而不是skip；
  • 随机挑选集合中的文档：添加一个键用来存储随机值，再进行过滤而不是随机跳过文档。

高级查询选项

绝大多数驱动程序都提供了辅助函数，用于向查询中添加各种选项。

常见的一些有用的选项

  • $maxscan : integer
    指定本次查询中扫描文档数量的上限；
  • $min : document
    查询的开始条件；
  • $max : document
    查询的结束条件；
  • $showDiskLoc : ture
    用于显示该条结果在磁盘上的位置。

获取一致结果

由于在查询过程中将文档取出进行处理后，可能会导致文档变大而移动到集合的末端，从而再遍历的时候又会被遍历到出现问题。

使用快照，查询将在”_id”的索引上遍历执行，这样可以保证每个文档只被返回一次。

游标声明周期

看待游标有两种角度：客户端的游标以及客户端游标表示的数据库游标。

释放游标的条件
  • 游标完成匹配结果的迭代时；
  • 客户端的游标已经不在作用域了；
  • 一个游标在10分钟内没有使用。

数据库命令

MongoDB中的命令被实现为一种特殊类型的查询，这些特殊的查询会在$cmd集合上执行。

数据库命令返回

  • 总会返回一个包含”ok”键的文档：键值为1，表示执行成功；否则表示执行失败；
  • 如果执行失败，还会在返回文档中包含”errmsg”
