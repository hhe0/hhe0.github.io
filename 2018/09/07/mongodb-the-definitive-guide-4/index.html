【MongoDB权威指南】读书笔记(四)

发表于 2018-09-07   |   更新于 2019-06-13 | 分类于 MongoDB读书笔记系列 | | 阅读次数

这部分主要介绍MongoDB的聚合。

聚合

介绍聚合框架、MapReduce和几个简单的聚合命令：count、count和group。

聚合框架

使用聚合框架可以对集合中的文档进行变换和组合：包括筛选(filtering)、投射(projecting)、分组(grouping)、排序(sorting)、限制(limiting)、跳过(skipping)。

示例：找出发表文章最多的那个作者

管道操作符

每个操作符都会接受一连串的文档，对这些文档做一些类型转换，最后将转换后的文档作为结果传递给下一个操作符（对于最后一个管道操作符，是将结果返回给客户端）。

$match

  • $match用于对文档集合进行筛选，之后就可以在筛选得到的文档子集上做聚合；
  • 不能以在$match中使用地理空间操作符；
  • 通常，在实际使用中应该尽可能将$match放在管道的前面位置。一是可以快速将不需要的文档过滤掉，以减少管道的工作量；而是如果在投射和分组之前执行$match，查询可以使用索引。

$project

  • 返回文档中想要/不想要的字段、重命名一个字段；
  • 重命名一个字段后需要显式指出将原字段排除，否则将返回该字段的两个副本；
  • 重命名字段后，新字段将不再享受原有字段的索引，因此应该尽量在修改字段名称之前使用索引。
管道表达式

在聚合框架中有几个表达式可用来组合或者进行任意深度的嵌套，以便创建复杂的表达式。

a. 数学表达式

常用的操作符的用法

表达式 含义
“$add” : [expr1[, expr2, …, exprN] 相加操作
“$subtract” : [expr1, expr2] 相减操作
“$multiply” : [expr1[, expr2, …, exprN] 相乘操作
“$divide” : [expr1, expr2] 相除操作
“$mod” : [expr1, expr2] 取余操作

b. 日期表达式

  • 常用的有”$year”, “$month”, “$week”, “$dayOfMonth”, “$dayOfWeek”, “$dayOfYear”, “$hour”, “$minute”, “$second”等；
  • 注意只能对日期类型的字段进行日期操作，不能对数值类型字段做日期操作

c. 字符串表达式

常用的操作符的用法

表达式 含义
“$substr” : [expr, starOffset, numToReturn] 截取一个字符串的numToReturn个字节
“$concat” : [expr1[, expr2, …, exprN] 连接操作
“$toLower” : expr 返回字符串的小写形式
“$toUpper” : expr 返回字符串的大小形式

d. 逻辑表达式
常用的操作符的用法

表达式 含义
“$cmp” : [expr1, expr2] 返回expr1-expr2的值作为比较结果
“$strcasecmp” : [string1, string2] 比较字符串，区分大小写（只对罗马字符有效）
“$eq”/“$ne”/“$gt”/“$lt”/“$lte” : [expr1, expr2] 返回比较的结果true/false
“$and” : [expr1[, expr2, …, exprN] and操作
“$or” : [expr1[, expr2, …, exprN]] or操作
“$not” : expr 取反操作
“$cond” : [booleanEpxr, trueExpr, falseExpr] 类似?:
“$ifNull” : [expr, replacementExpr] expr为空返回后者，否则返回expr

实例

加入有个教授想通过某种比较复杂的计算为学生加分：出勤率占10%，日常测试占30%，期末考试占60%（如果是老师最宠爱的学生，那么分数就是100）

$group

将文档依据特定字段的不同值进行分组。

分组操作符

允许对每个分组进行计算。

常用的操作符

表达式 含义
“$sum” : value 对于分组中的每一个文档，将value与计算结果相加
“$avg” :value 返回每个分组的平均值
“$max” : expr 返回分组内的最大值
“$min” : expr 返回分组内的最小值
“$first” : expr 返回分组的第一个值，排序后有意义
“$last” : expr 返回分组的最后一个值，排序后有意义

如果数组时有序的，那么$first和$last比$min和$max效率高，否则$min和$max效率高。

$unwind

将数组中的每一个值拆分为单独的文档。

得到特定用户的所有评论（只需要评论，不需要返回评论所属的文章）

$sort

  • 如果要对大量的文档进行排序，强烈建议在管道的第一阶段进行排序，这时的排序操作可以使用索引；
  • 可以在排序中使用文档中实际存在的字段，也可以使用在投射时重命名的字段。

$limit

$limit会接受一个数字n，返回结果集中的前n个文档。

$skip

$skip也是接受一个数字n，丢弃结果集中的前n个文档，将剩余文档作为结果返回。

使用管道

  • 应该尽量在管道的开始阶段（执行”$project”、”$group”、”$unwind”操作之前）就将尽可能多的文档和字段过滤掉；
  • 如果MongoDB发现某个聚合操作占用了20%以上的内存，这个操作就会直接输出错误。

MapReduce

MapReduce可以表达任意复杂的逻辑，代价是速度非常慢，不应该用在实时的数据分析中。

MapReduce的几个步骤

  • 映射：将操作映射到集合中的每个文档。这个操作要么“无作为”，要么“产生一些键和X个值”；
  • 洗牌：按照键分组，并将产生的键值组成列表放到对应的键中；
  • 化简：把列表中的值化简成一个单值，返回该值；
  • 重复洗牌和化简，直到每个键的列表只有一个值为止。

MapReduce返回文档的参数

  • “result” : “tmp.mr.mapreduce_1266787811_1”
    这是存放MapReduce结果的集合名。这是个临时集合，MapReduce的连接关闭后它就被自动删除了。
  • “timeMillis” : 12
    操作花费的时间，单位是毫秒
  • “counts” : {…}
    主要用于调试，包含三个键
    • “input” : 6
      发送到map函数的文档个数
    • “emit” : 14
      在map函数中emit被调用的次数
    • “output” : 5
      结果集合中的文档数量

MongoDB和MapReduce

MapReduce命令其他可选的键

表达式 含义
“finalize” : function 可以将reduce的结果发送给这个键，作为整个处理过程的最后一步
“keeptemp” : boolean 如果值为true，那么在连接关闭时会将临时结果集合保存下来，否则不保存
“out” : string 输出集合的名称
“query” : document 在发往map函数前，先用指定条件过滤文档
“sort” : document 在发往map前先给文档排序
“limit” : integer 发往map函数的文档数量的上限
“scope” : document 可以在js代码中使用的变量
“verbose” : boolean 是否记录详细的服务器日志

聚合命令

MongoDB为在集合上执行基本的聚合任务提供了一些命令。一些简单的命令例如count、distinct可以不使用聚合框架而简化为普通命令使用。

count

  • count是最简单的聚合工具，用于返回集合中的文档数量；
  • 增加查询条件会使count变慢。count可以使用索引，但是索引并没有足够的元数据供count使用，所以不如直接使用查询来得快。

distinct

distinct用来找出给定键的所有不同值。使用时必须指定集合和键。

需要获取集合里面所有不同的键时，无法使用distinct，可以用MapReduce实现。

group

使用group可以执行更复杂的聚合。先选定分组所依据的键，而后MongoDB就会将集合依据选定键的不同值分成若干组。然后可以对每一个分组内的文档进行聚合，得到一个结果文档。

命令相关的键
命令 含义
“ns” : “stocks” 指定要分组的集合
“key” : “day” 指定文档分组依据的键
“initial” : {“time” : 0} 每一组reduce函数调用中的初始”time”值，会作为初始文档传递给后续过程。
“$reduce” : function(doc, prev){…} 在集合内的每个文档上执行。doc表示当前文档，prev表示累加器文档
