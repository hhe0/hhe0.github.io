【go】map

发表于 2019-08-02   |   更新于 2019-08-02 | 分类于 go学习笔记 | | 阅读次数

散列表是设计精妙、用途广泛的数据结构之一。它是一个拥有键值对元素的无序集合点。

map的特点

  • 在map中，键的值是唯一的，键对应的值可以通过键来获取、更新或移除；
  • 无论这个散列表有多大、这些操作基本上是通过常量时间的键比较就可以完成；
  • map中所有的键都拥有相同的数据类型，同时所有的值也都拥有相同的数据类型。

map元素的地址

  • 无法获取map元素的地址的一个原因是map的增长可能会导致已有元素被重新散列到新的存储位置，这样就可能使得获取的地址无效。

map的零值

  • map的零值是nil；
  • 大多数的map操作都可以安全地在map的零值nil上执行，包括查找元素，删除元素，获取map元素个数，执行range循环，因为这和空map的行为一致，但是向零值map中设置元素会导致错误；
  • 通过下标访问map中的元素总是会值。如果键在map中，会得到对应的值；否则得到相应的零值。

map的比较

  • map不可比较，唯一合法的比较就是和nil比较。

map的遍历

  • 与数组、slice遍历的方式相同；
  • map中元素的迭代顺序是不固定的，不同的实现方法会使用不同的散列算法，得到不同的元素顺序。

如果需要按照一定的顺序来遍历，必须显式地给键排序。
